# VILLAE ROMANAE — инструкции проекта

Статический Leaflet-атлас античных памятников. Без сборки: HTML + Leaflet 1.9.4 (CDN) +
ванильный JS (глобалы) + CSS. Хостинг: GitHub Pages + Cloudflare → villae-romanae.com.

## Регионы и файлы
- **Израиль, виллы:** `index.html` / `app.js` / `data.js` (глобал `SITES`)
- **Израиль, арх. объекты:** `sites.html` / `sites.js` / `sites-data.js` (`ARCH_SITES`)
- **Причерноморье:** `pontus.html` / `pontus.js` / `regions/pontus/{sites-data,images}.js` (`PONTUS_*`)
- **Таймлайн:** `timeline.html`
- Общее: `styles.css`, `periods.js`. Ресёрч и доки: `research/`, `docs/`

## Модели и субагенты (контроль расходов)
Главный рычаг экономии — правильно раскидывать работу по моделям:
- **Лёгкую и параллельную работу делегируй субагентам на Sonnet 5** (`model: sonnet`):
  ресёрч источников, поиск по файлам/коду, сбор инвентаря памятников, разведка вширь,
  поиск фоток. Есть готовая роль `researcher` (`.claude/agents/researcher.md`).
  Sonnet 5 на интро-прайсе ($2/$10 до 31.08.2026) — ~в 2.5× дешевле Opus.
- **Opus (основной цикл) — только для тяжёлого:** реализация кода, злая отладка,
  интеграция, финальный синтез.
- **Не разворачивай веер субагентов ради тривиального** (одно чтение файла, мелкая правка):
  у каждого субагента свой кэш, он не шарится с основным — веер теряет экономию.
- **Субагенты НЕ плодят своих субагентов.** Веер запускает только основной цикл (я), и
  контролируемым числом (обычно 1, максимум несколько под конкретные зоны). Субагент обязан
  вернуть данные сам, без рекурсивного делегирования — иначе расход неконтролируем
  (проверено больно: один «микро»-агент развернул шесть, спалил ~460k токенов).
- **Для ресёрча/поиска бери агента БЕЗ инструмента Agent** — `Explore` (read-only, физически
  не может плодить субагентов) или роль `researcher`. НЕ `general-purpose`: у него `*`, включая
  сам Agent → может рекурсивно веерить и лезет в лишние MCP-инструменты (один спалил 167k / 136
  вызовов на ~11 фоток и зачем-то дёрнул mark_chapter). Узкий тулсет = фокус, дешевле, структурно
  нельзя развернуть веер.
- Держи сессию непрерывной: паузы >5 мин протухают промпт-кэш и заставляют переписывать
  контекст по ×1.25 вместо чтения по ×0.1.

## Новые регионы (напр. скифы)
Движок (карта, регион-свитч, event-driven попапы, автодекластеринг лейблов, стили,
кликабельная библиография) уже построен и отлажен — **переиспользуй как есть.**
Новый регион = новый датасет (`*-data.js`) + фотки (`images.js`) + проводка региона.
Интерактив не перестраивай — там и был зарыт основной расход прошлых регионов.

## Операционка (пропустишь — словишь дорогой передёбаг)
- **После любой правки CSS/JS бампай `?v=N`** во всех `.html`, иначе Cloudflare отдаёт старое.
- Проверяй на localhost (preview на :8137) до деплоя.
- **Деплой автоматический (Дмитрий дал постоянное добро 2026-07-01):** как только правка
  проверена на localhost — сразу `git add <файлы> && git commit && git push origin main`,
  без доп. вопросов. `main` = прод (GitHub Pages + Cloudflare → villae-romanae.com).
  Коммить только свои файлы; НЕ коммить `.DS_Store` и прочий мусор. Пуш — с `dangerouslyDisableSandbox`.
- Попапы — только event-driven (`bindPopup` один раз + `popupopen`/`popupclose`).
  Никогда не вызывай `openPopup` в обработчике клика по маркеру — ломается на iOS.

Глубокие детали схемы данных, цветов и прошлых фиксов — в памяти (`villae-romanae-project`).
