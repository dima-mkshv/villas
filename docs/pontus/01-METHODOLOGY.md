# 01 — Методология: отбор, ранжирование, поля, фильтры, маркеры

Как мы решаем, что попадает в атлас, как это оцениваем и какими полями описываем.
Источники подробно — в [04-SOURCES.md](04-SOURCES.md); схема записи — в [03-DATA-SCHEMA.md](03-DATA-SCHEMA.md).

## 1. Что включаем (типы объектов)

Максимальный охват → берём все классы античных памятников региона:

- **Города / городища** (полисы и крупные центры): Фанагория, Гермонасса, Горгиппия…
- **Поселения** (рядовые, сельская округа — хора): Патрей, Тирамба, «Волна», Мысхако…
- **Крепости / укрепления** (в т.ч. позднеантичные и византийские).
- **Некрополи** (курганные и грунтовые; склепы с росписью — напр. склеп Геракла в Горгиппии).
- **Святилища / культовые места**.
- **Подводные объекты** (затопленные части Фанагории, Акры-аналоги) — с пометкой.

**Не включаем** (вне темы): дольмены бронзового века, средневековые черкесские/генуэзские и
позднее, если нет античного слоя. Граница периода — см. [02-PERIODIZATION.md](02-PERIODIZATION.md).

## 2. Критерии отбора

Объект попадает в атлас, если выполнено хотя бы одно:
1. **Идентифицирован** в науке (есть раскопки/разведки и публикации), **или**
2. **Назван в античных источниках** (Страбон, перипл Псевдо-Арриана, Птолемей) — даже если
   локализация спорна (тогда отметка «локализация предположительна»), **или**
3. **Музеефицирован / экспонируется** (музей-заповедник, открытый показ).

Чисто гипотетические точки без привязки — только в библиотеку, не на карту.

## 3. Ранжирование (значимость)

В Израиле осью был «рейтинг вида». Здесь он не подходит. Предлагаемая ось — **значимость**
(`significance` 1–5), составная экспертная оценка по 4 факторам:

| Фактор | Что учитываем |
|---|---|
| Историческая роль | столица/полис vs рядовое поселение; роль в Боспоре |
| Изученность | объём раскопок, корпус находок, публикации |
| Что видно сейчас | музеефикация, наличие экспозиции/руин на поверхности |
| Сохранность | насколько уцелел культурный слой/объекты |

✅ **Решено:** `significance` 1–5 вводим (звёзды, для подсветки топ-объектов), **но это НЕ
обязательный фильтр** — пользователь не отсекает объекты «по рейтингу». Главные, важные для
посетителя фильтры — **доступность** (туристический/закрытый) и **сохранность**, плюс эпоха и
тип. `significance` — только для сортировки/акцента и ★, не для скрытия.

## 4. Объективные метрики объекта

Заказчик просил размер / сохранность / площадь — выносим в отдельные поля:

- **`areaHa`** — площадь памятника в гектарах (напр. Фанагория >65 га; для подводной части —
  примечание). Числом, где известно; иначе `null` + словесно в `sizeNote`.
- **`sizeClass`** — категория размера для фильтра: `metropolis` (>40 га) · `city` (10–40) ·
  `town` (2–10) · `small` (<2) · `unknown`.
- **`preservation`** — сохранность, шкала: `excellent` · `good` · `partial` · `poor` ·
  `lost` (застроено/размыто/срыто). Со словесным `preservationNote`.
- **`excavationStatus`** — `active` (копают сейчас) · `past` · `surveyed` (разведки) ·
  `unexcavated`.

Все «сырые» обоснования цифр (откуда площадь, чьи оценки, разночтения) — в библиотеку.

## 5. Статус доступа (ключевой фильтр для простого пользователя)

Заказчик хочет «туристический / закрытый археологический». Делаем поле **`access`**:

| Значение | Смысл | Пример |
|---|---|---|
| `museum` | Музей-заповедник / музеефицировано, открыто туристу | Фанагория, Горгиппия |
| `open` | Свободный доступ к руинам/городищу, без инфраструктуры | многие городища Тамани |
| `restricted` | Действующий раскоп / охранная зона / по согласованию | активные экспедиции |
| `closed` | Закрыто / на территории части / застроено / под водой | подводная Фанагория |
| `lost` | Физически не сохранилось, только точка на карте | срытые/размытые |

**Фильтр на сайте** даёт пользователю минимум два переключателя:
- **«Туристические»** (`museum` + `open`) — куда реально можно поехать.
- **«Закрытые / археологические»** (`restricted` + `closed` + `lost`) — для интересующихся.

Плюс к этому фильтры: **эпоха** (легенда-эпохи), **тип объекта**, и (опц.) **размер** /
**сохранность**. Набор фильтров финализируем в Phase 3.

## 6. Маркеры

Как в Израиле, но семантика под регион:

- **Цвет = эпоха** (главная эпоха объекта; все эпохи — тегами в карточке). Палитра и эпохи —
  [02-PERIODIZATION.md](02-PERIODIZATION.md).
- **Форма = тип объекта** (переиспользуем механизм круг/квадрат/ромб):
  🟡 предложение — **круг** = город/поселение, **квадрат** = крепость, **ромб** = некрополь,
  **треугольник** = святилище. (Уточним при вёрстке.)
- **★ значок** — для топ-значимости и/или музеефицированных «маст-си».

## 7. Двуязычие и названия

- `name` = { ru, en } — современное/принятое название.
- `nameAncient` = античное имя (греч. и/или лат.): *Phanagoreia*, *Hermonassa*, *Gorgippia*,
  *Bata*, *Toricos*, *Nikopsis*… — показываем в карточке (важно для темы).
- Современная привязка (нас. пункт): пос. Сенной, ст. Тамань, Анапа…

## 8. Качество и честность данных

- **Локализация спорная** (Бата, Торик, Никопсия и др. периплов) → флаг `localizationCertain:
  false` + разбор версий в библиотеке. На карте — приблизительная точка с пометкой.
- **Координаты** — из музея/ИА РАН/публикаций/викиданных; проверяем, что на суше/в нужной бухте.
- **Разночтения** дат и атрибуций фиксируем в библиотеке со ссылкой на источник, на сайт
  выносим консенсус.
- Приоритет источников — см. иерархию в [04-SOURCES.md](04-SOURCES.md).
