# 05 — Библиотека исследований (для будущих агентов)

Бриф п.6: нужна **отдельная база того, что «наковыряли»** — чтобы следующий агент мог
**рыться, добавлять и исправлять**, и чтобы **не вся** инфа висела на сайте.

Принцип: **сайт = кураторская витрина, библиотека = полное досье.** Сайт держит выжимку
(карточка), библиотека — источники, разночтения, обоснования цифр, сырьё и журнал правок.

## 1. Где живёт

```
research/
  pontus/
    _index.md                 реестр объектов + статус + покрытие по районам
    _sources.md               мастер-библиография региона (все источники разом)
    _localization-disputes.md  спорные локализации (Бата, Торик, Никопсия…)
    _conventions.md           правила (этот файл — короткая выжимка-инструкция)
    phanagoria.md             досье объекта
    hermonassa.md
    gorgippia.md
    …
```

Связь с сайтом: запись в датасете имеет `_libRef: "research/pontus/<id>.md"` — мост от
карточки к полному досье. Поля с `_` на сайт не рендерятся.

## 2. Шаблон досье объекта (`research/pontus/<id>.md`)

```markdown
---
id: phanagoria
status: published            # draft | researched | published | needs-review
confidence: high             # high | medium | low
verified: 2026-06-30 / agent-name
on_site: true                # вынесено ли на карту
---

# Фанагория / Phanagoreia

## Идентификация
Что это, тип, античное имя, метрополия-основатель, современная привязка.

## Локализация
Координаты, уверенность. Если спорно — **версии** с источниками и кто за какую.

## Хронология
Эпохи, основание/гибель, обоснование дат (со ссылками). Разночтения.

## Метрики (размер / площадь / сохранность)
- Площадь: 65+ га — источник: музей-заповедник / публикация X.
- Сохранность: … (что уцелело, что под водой/застроено).
- Изученность: годы раскопок, кто, объём.

## Раскопки и находки
Ключевые находки, отчёты, активные экспедиции.

## Источники
- Античные: Страбон XI.2.10; Псевдо-Скилак 72; …
- Эпиграфика: КБН …; INBS …
- Современные: Кузнецов В.Д. …; ИА РАН отчёты …

## Что на сайте / что осталось здесь
Какие поля ушли в карточку (выжимка), какие детали — только тут.

## Открытые вопросы / TODO
Что проверить, уточнить, перепроверить.

## Журнал
- 2026-06-30 agent-X: создано досье, выверена площадь.
```

## 3. `_index.md` — реестр и покрытие

Таблица всех объектов с колонками: `id`, район (Тамань/Анапа/Новороссийск/Геленджик/Сочи),
тип, эпохи, `status`, `confidence`, `on_site`, ссылка на досье. Сразу видно прогресс и дыры
(«по Геленджику только Торик, добрать хору»). Здесь же — счётчики покрытия.

## 4. Рабочий процесс для агента

**Добавить объект:**
1. Создать `research/pontus/<id>.md` по шаблону, заполнить из источников (с приоритетом по
   [04-SOURCES.md](04-SOURCES.md)).
2. Если объект годен для карты (критерии — [01-METHODOLOGY.md](01-METHODOLOGY.md)) — перенести
   **выжимку** в `regions/pontus/sites-data.js` по [схеме](03-DATA-SCHEMA.md), проставить `_libRef`.
3. Обновить `_index.md` (статус, покрытие).

**Исправить / уточнить:**
1. Правка досье + запись в «Журнал» (кто/когда/что/источник).
2. Синхронизировать изменённые поля в `sites-data.js`.
3. Обновить `confidence`/`status` в шапке досье и в `_index.md`.

## 5. Правила (конвенции)

- **Не выдавать гипотезу за факт.** Спорное → `confidence: low/medium` + версии в досье,
  на сайте — «предположительно».
- **Каждая цифра — со ссылкой** (площадь, дата). Без источника — помечать как оценку.
- **Источники — по иерархии** ([04-SOURCES.md](04-SOURCES.md)); Wikipedia — не для спорного.
- **Античные названия** обязательны (греч./лат.).
- **Журнал правок** ведём всегда — чтобы следующий агент видел историю и не переоткрывал спорное.
- **Сайт не раздуваем**: длинные разборы, цитаты целиком, разночтения — в библиотеку; в
  карточку — выжимку и `bibliography`-ссылки.

## 6. Зачем так

- Преемственность: новый агент за 1 файл (`_index.md`) видит, что сделано и что осталось.
- Воспроизводимость: видно, **откуда** каждая цифра и атрибуция.
- Чистый сайт: пользователь не тонет в науке, но при желании уходит по ссылкам/в библиотеку.
- Безопасные правки: журнал + статусы не дают «молча испортить» выверенные данные.
