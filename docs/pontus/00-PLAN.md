# 00 — План и архитектура

## 1. Видение

Расширить существующий атлас (`villas`, ныне про римский Израиль) до **мультирегионального**
и добавить второй регион — **античное Северное Причерноморье (российский сектор)**:
Таманский полуостров и Черноморское побережье Краснодарского края от Анапы до Сочи.

Тема региона: **греческая колонизация и Боспорское царство** — от основания колоний
(сер. VI в. до н.э.) через эллинистический расцвет и римский протекторат до византийской
Таматархи. Фокус на греках, но эпохи показываем все (см. [02-PERIODIZATION.md](02-PERIODIZATION.md)).

## 2. Что переиспользуем из `villas` (движок готов)

Текущий код почти полностью переносится — это его вторая «загрузка данными»:

- **Leaflet-карта**, базовые подложки (рельеф/спутник/светлая), маркеры-формы, попапы.
- **Тёмная тема**, навбар, RU/EN (`nav.js`, событие `vr:langchange`, `?lang=` в URL).
- **Модель «эпоха = цвет»**: `periods.js` → для Причерноморья делаем свой набор эпох.
- **Таймлайн** (`timeline.html` + `timeline.js`) — параметризуется набором эпох региона.
- **Карта объектов** (`sites.html` + `sites.js`) — фильтр-легенда по эпохам, deep-links.
- **Кэш-бастер `?v=`**, деплой на GitHub Pages + Cloudflare (опыт уже есть).

## 3. Что меняем — рефакторинг в мультирегион

Цель: один движок, N регионов, переключатель.

```
/regions/
  israel/      data.js, sites-data.js, periods.js   (перенос текущих)
  pontus/      data.js, sites-data.js, periods.js   (новый)
regions.js     реестр регионов: id, название RU/EN, центр карты, zoom, список датасетов,
               набор эпох, базовый домен/путь
```

Ключевые правки (Phase 1):
- Вынести жёстко зашитые сейчас вещи (центр карты `fitBounds`, набор `PERIODS`, тексты) в
  **конфиг региона**. Сейчас периоды и данные глобальны — параметризуем по региону.
- Навбар: добавить **переключатель региона** (Израиль / Причерноморье), хранить выбор в
  `localStorage` (`vr_region`) и в URL (`?region=pontus`), по аналогии с `?lang=`.
- Каждый регион грузит свои `periods` + датасеты; легенда/таймлайн/карта читают активный регион.

✅ **Бренд (решено).** Пока оставляем **«VILLAE ROMANAE»** как зонтичный; регион подписываем
рядом (напр. «VILLAE ROMANAE · Причерноморье»). Переименование — потом, если захочется.

✅ **Домен (решено).** Один домен **`villae-romanae.com`** + параметр **`?region=pontus`**
(по аналогии с `?lang=`). Отдельный домен не заводим.

## 4. Чем Причерноморье отличается от Израиля (влияет на движок)

| Аспект | Израиль | Причерноморье | Следствие для кода |
|---|---|---|---|
| Главная ось | рейтинг вида (1–5) | эпоха + значимость | убрать «вид» из региона, ввести метрики (площадь/сохранность) |
| Маркер-цвет | период | эпоха | тот же механизм, свой набор эпох |
| Доступ | почти всё — нац.парки | **много закрытых раскопов** | важен фильтр «туристический / закрытый» (новый, см. методологию) |
| Источники | Wikipedia/INPA | **античные авторы + эпиграфика + раскопочные отчёты** | расширить карточку библиографией |
| Локализация | известна точно | **часть спорна** (Бата, Торик…) | хранить уверенность атрибуции в библиотеке |

## 5. Roadmap (фазы)

- **Phase 0 — Документация (сейчас).** Этот пакет. Ревью заказчиком. ⬅️ мы здесь
- **Phase 1 — Рефакторинг движка в мультирегион.** Конфиг регионов, переключатель, перенос
  Израиля в `/regions/israel/` без потери функционала. Деплой, проверка что Израиль цел.
- **Phase 2 — Данные Причерноморья.** По методологии: research → `research/`-библиотека →
  кураторские `pontus/sites-data.js` + `periods.js`. Итерациями по районам (Тамань → Анапа →
  Новороссийск → Геленджик → Сочи).
- **Phase 3 — Фичи региона.** Фильтры доступа/типа/сохранности, расширенная карточка с
  библиографией и цитатами источников (Страбон и пр.), таймлайн эпох Причерноморья.
- **Phase 4 — Полировка и деплой.** Мобилка, OG, кэш-бастер, аналитика.

## 6. Границы MVP (Phase 2 минимум)

«Минимально ценно» = карта Причерноморья с эпохами-маркерами, ~15–20 знаковых объектов
(Фанагория, Гермонасса, Кепы, Горгиппия, Бата, Торик, Никопсия…) с фильтром доступа и
карточками с библиографией. Максимальный охват (десятки) добиваем итерациями в Phase 2/3,
сырьё копится в библиотеке параллельно.

## 7. Нефункциональные принципы

- **Без сборки** — статика, как сейчас.
- **Данные отделены от движка** — регион = набор JS-датасетов + конфиг.
- **Кураторская выжимка на сайте, полнота — в библиотеке** (см. [05-RESEARCH-LIBRARY.md](05-RESEARCH-LIBRARY.md)).
- **Двуязычность RU/EN**, античные названия (греч./лат.) — отдельным полем.
- **Честность атрибуций** — спорные локализации помечаем, не выдаём гипотезу за факт.
